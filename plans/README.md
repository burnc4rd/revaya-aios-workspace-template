# plans/

> Implementation plans created by `/create-plan`.
>
> Plans are dated markdown files that capture context, rationale, and step-by-step tasks needed to execute a change. Every significant build starts with a plan.
>
> **Naming convention:** `YYYY-MM-DD-descriptive-name.md`
>
> **Workflow:** `/create-plan [request]` → `/challenge [plan-path]` (mandatory) → `/implement [plan-path]`
>
> Subdirectories are optional — organize by project if you have multiple active builds.
